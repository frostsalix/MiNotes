# 2026-05-19

## AGP 兼容性问题修复
- **问题**: `ApplicationExtensionImpl$AgpDecorated_Decorated cannot be cast to BaseExtension` — AGP 9.0.0 + Gradle 9.5.1 + Kotlin 2.1.20 工具链过于激进
- **修复**: 降级到稳定组合
  - Gradle: 9.5.1 → 8.11.1
  - AGP: 9.0.0 → 8.7.3
  - Kotlin: 2.1.20 → 2.0.21
- **清理**: 全局 Gradle cache、项目 .gradle、app/build 均已删除
- **注意**: 项目没有 gradlew 脚本，需要 Android Studio 重新 Sync 来拉取新 Gradle 版本
- **补充**: Gradle 官方源下载超时，已将 distributionUrl 换为腾讯云镜像 `https://mirrors.cloud.tencent.com/gradle/`

## 黑屏问题修复（补充 - 第二轮）
- **真实根因（重新确认）**: Activity 继承原生 `Activity`，主题却使用 `Theme.MaterialComponents.Light.NoActionBar.Bridge`（需要 `AppCompatActivity`），导致主题不兼容崩溃黑屏
- **修复内容**:
  1. `NotesListActivity`、`NoteEditActivity`、`AlarmAlertActivity` 全部改为继承 `AppCompatActivity`
  2. 对应 `import android.app.Activity` → `import androidx.appcompat.app.AppCompatActivity`
  3. `AlertDialog` import 改为 `androidx.appcompat.app.AlertDialog`
  4. `NoteTheme` parent 改为 `Theme.MaterialComponents.Light.NoActionBar`（去掉 Bridge 后缀）
  5. 依赖升级: `appcompat:1.3.1→1.7.0`, `material:1.4.0→1.12.0`（旧版在 API36 有兼容性问题）
